# R-A2 · Reflection / feedback routing

**Reframe:** the reflection *plumbing* already exists in skeleton — an until-loop already
re-prompts itself with the prior failure and can frame it as Reflexion/Self-Refine, and the
verifier already routes its critique into the next attempt. A2 is: **make that routing
eval-driven (the A1 `critique`), measured, and durable across runs** — not build reflection
from zero.

---

## Current state (from §2, §3, §5)

- **Intra-run reflection exists.** On a failed attempt, `adaptive_retry` stashes the score's
  errors as `last_error` framed by a `SelfPromptStrategy`
  (`Direct/Reflexion/SelfRefine/PlanThenAct`) that prepends a reflection preamble, and the
  strategy can escalate one rung per failure (`run_loop.rs:426-442`;
  `crates/lopi-core/src/self_prompt.rs`). The next attempt's planning prompt consumes
  `last_error` via `gather_seed()` (`runner/seed.rs`).
- **Verifier critique already routes forward.** A FAIL verdict's `fix_hints` are appended to
  `task.constraints` (deduped) and picked up by the next attempt's seed
  (`verifier_runner.rs:73-82`). This is Reflexion, already wired — but only from the finalize
  verifier, only on already-passing-then-rejected work.
- **Durable learning surfaces exist but are thin.** `lessons` (free-text, repo-scoped,
  **write silently gated at score ≥ 0.6**, `lessons.rs:36-64`); `patterns` (goal-keyword
  fingerprint + rolling success rate, `patterns.rs:167-218`); `insert_postmortem_pattern`
  on terminal failure (`patterns.rs:139`, called from `run_postmortem_if_configured`).
- **Gaps:** critique today comes from the heuristic score or the finalize verifier — **not**
  from a tiered eval verdict (A1 doesn't exist yet). No per-iteration critique history. No
  measured proof that reflection beats blind retry. `kohaku`/vector recall is vaporware
  (§5) — durable learning must ride `MemoryStore`.

## Prior art mapped onto lopi's assets

| Prior art | lopi already has | A2 adds |
|---|---|---|
| Reflexion (verbal self-feedback into next attempt) | `SelfPromptStrategy::Reflexion` + `last_error` routing (`run_loop.rs:440`) | feed the **A1 `EvalOutcome.critique`** in, not just the raw score errors |
| Self-Refine (critique → revise same artifact) | in-place `claude.fix(errors)` (`run_loop.rs:381`) + `SelfRefine` framing | route eval `per_check` gaps into the fix prompt |
| Verifier critique injection | `fix_hints → constraints` (`verifier_runner.rs:77`) | generalize to any tier's gaps; dedupe across attempts |
| Episodic recall / "compounding loop" | `lessons` + `patterns` + `find_similar_patterns` (`patterns.rs:80`) | write a **reflection** learning at run end; recall it at seed time |

## Design + recommendation

### 1. Route the A1 verdict's critique into the next iteration (intra-run)

When `TieredEvaluator` returns `Verdict::Fail`, its `EvalOutcome.critique` (flattened
`fix_hints`/`gaps` from `per_check`) becomes the reflection payload. Feed it through the
**existing** `SelfPromptStrategy` framing at `run_loop.rs:440` — i.e. replace/augment the
`base_failure` string with the structured eval critique. This reuses the whole existing
reflect-and-retry path; A2's new code is "eval critique → strategy input," not a new loop.

Keep the escalation ladder (`effective_strategy`, `run_loop.rs:440`): Direct → Reflexion →
Self-Refine → Plan-Then-Act as failures accumulate.

### 2. Persist a per-iteration critique trail

A1 already persists `EvalOutcome` per attempt (`eval_outcomes` table). A2 adds nothing to
storage here — it *reads* that trail so iteration N's prompt can show "what the evaluator
said about attempts 1..N-1," not just N-1. This is the difference between one-step Reflexion
and a running reflection memory within a task.

### 3. Durable cross-run learning (the compounding loop)

At run end (`run_one` post-run seam, `pool/run_loop.rs:473-495`, next to `mine_patterns`),
write a **reflection lesson**: a distilled "what worked / what the evaluator kept rejecting"
derived from the `eval_outcomes` trail. Reuse `save_lesson()` — but **fix the silent 0.6
gate** (`lessons.rs:36-64`): a hard-won *failure* lesson is exactly the ≥0.6-score case that
never happens, so gate on "run produced a durable insight," not on final score, or log the
skip (`tracing::warn!`) instead of silently dropping it (the CLAUDE.md "no silent failures"
rule applies). At seed time, `gather_seed()` already pulls `find_similar_patterns` and
`load_lessons` — A2 just ensures reflection lessons are among them.

**Recommendation:** ship intra-run first (route A1 critique through the existing self-prompt
path — small), then durable learning (reflection lesson at the post-run seam + un-silence
the lessons gate — small). Both reuse existing surfaces. Size: **S–M**, gated on A1.

## Pre-registered BUILD kill-test

> **Pre-register (fix margin before running):**
> 1. **Visibility:** on a task that fails attempt 1, attempt 2's planning prompt *provably
>    contains* the evaluator's critique from attempt 1 (assert the prompt string includes the
>    `fix_hints`). Binary pass/fail.
> 2. **Measured lift:** on a fixed set of **N ≥ 15** tasks that fail on the first blind try,
>    **reflect-and-retry beats blind-retry** in first-follow-up pass rate by a
>    **pre-registered margin of ≥ 15 percentage points**, run paired (same tasks, same
>    seeds) to control variance. If the lift is < 15pp or not statistically distinguishable
>    from noise on a paired sign test, A2 as designed is not proven — report the honest
>    number and revisit the critique format before shipping.

The margin is pre-registered here so it can't be moved after seeing results. The *measured
comparison* is the deliverable, not the feature.

## Biggest risk

**Reflection that doesn't move the needle** — the literature shows Reflexion helps only when
the critique is specific and actionable; a vague "it failed, try again" critique adds tokens
and no lift, and can even anchor the model on a wrong fix. Mitigation: the A1 judge already
emits *specific* `gaps`/`fix_hints` (the probe showed it names the exact failure mode), so
feed those verbatim, not a generic summary. Measure (kill-test #2) before believing it.
Secondary risk: reflection lessons polluting memory with low-signal noise — keep the write
gated on a real insight and make lessons pruneable.

## Open questions

1. Structured critique (per-check gaps as list) vs prose framing — which does the worker
   act on better? (A/B inside the build kill-test.)
2. How many prior attempts' critiques to carry forward before the prompt bloats — full trail
   vs last-K? (Interacts with `lopi-context` eviction.)
3. Should a durable reflection lesson be repo-scoped (like `lessons`) or goal-pattern-scoped
   (like `patterns`)? Both recall paths exist; pick by what generalizes.
4. Do we ever *reflect the ratchet* — i.e. tell the worker "attempt 3 regressed vs attempt
   2's score"? That couples A2 to A3's score history; decide the ordering.
